## let vs. const

I was absent on Thursday due to illness, but reviewed the slides afterward and learned about `let` and `const` — two ways to store values in JavaScript.

- **`let`** — use when the value will change (e.g. a score going up)
- **`const`** — use when the value stays the same (e.g. a fixed rate)

```js
let score = 0;
score = 10; // works fine

const taxRate = 0.07;
taxRate = 0.08; // ERROR — can't change a const
```

A simple rule: use `const` by default, and switch to `let` only if the value needs to change.

HTML Reference Guide

A beginner-friendly overview of headings, common element types, and semantic HTML.

Headings

HTML provides six levels of headings, from <h1> (the most important) down to <h6> (the least important). Headings create a hierarchy that helps users and search engines understand the structure of your page.

Use only one <h1> per page. It should describe the main topic. Then use <h2> for major sections, <h3> for sub-sections, and so on.

h1 — Main Page Title

h2 — Section Heading

h3 — Sub-section Heading

h4 — Detail Heading

h5 — Minor Heading
h6 — Smallest Heading

Code:

<h1>Main Page Title</h1> <h2>Section Heading</h2> <h3>Sub-section Heading</h3> <h4>Detail Heading</h4> <h5>Minor Heading</h5> <h6>Smallest Heading</h6>

Common HTML Element Types

Paragraphs & Line Breaks

The <p> tag wraps a block of text into a paragraph. Use <br> to insert a single line break within a paragraph.

<p>This is a paragraph of text.</p> <p>First line.<br>Second line in the same paragraph.</p>

Links

The <a> (anchor) tag creates clickable hyperlinks. The href attribute sets the destination URL.

Visit Babson College
<a href="https://www.babson.edu">Visit Babson College</a>

Images

The <img> tag embeds an image. It is self-closing (no end tag). The src attribute points to the image file, and alt provides a text description for accessibility.

<img src="images/photo.jpg" alt="A description of the image">

Lists

<ul> creates an unordered (bulleted) list. <ol> creates an ordered (numbered) list. Each item inside uses <li>.

Unordered List:

  • HTML
  • CSS
  • JavaScript

Ordered List:

  1. Plan
  2. Build
  3. Launch
<ul> <li>HTML</li> <li>CSS</li> </ul> <ol> <li>Plan</li> <li>Build</li> </ol>

Div & Span (Generic Containers)

<div> is a block-level container used to group elements together (takes up the full width). <span> is an inline container used to style a small piece of text without breaking the flow.

<div> <p>This paragraph is inside a div.</p> </div> <p>This word is <span style="color:red;">red</span>.</p>

Semantic HTML

Semantic elements clearly describe their meaning to both the browser and the developer. Instead of using generic <div> tags for everything, semantic tags tell you what kind of content is inside.

Why does it matter? Semantic HTML improves accessibility (screen readers understand your page), SEO (search engines rank structured content higher), and readability (other developers can follow your code more easily).

Page Structure Elements

TagDescription
<header>Introductory content at the top of a page or section. Typically contains a logo, site title, and navigation.
<nav>A set of navigation links. Use it for your main menu, sidebar links, or breadcrumbs.
<main>The primary content of the page. There should only be one <main> per page.
<footer>Content at the bottom of a page or section. Usually contains copyright info, contact links, or legal text.

Content Grouping Elements

TagDescription
<section>A thematic group of content, usually with its own heading. Example: an "About" section or "Contact" section.
<article>A self-contained piece of content that could stand alone (e.g., a blog post, news story, or comment).
<aside>Content that is related but separate from the main content — like a sidebar, pull quote, or advertisement.
<figure>Self-contained content like an image, diagram, or code snippet, often paired with <figcaption>.
<figcaption>A caption or description for the content inside a <figure>.

Full Semantic Page Example

Here is how a typical page is structured using semantic elements:

<header> <nav> <a href="/">Home</a> <a href="/about">About</a> </nav> </header> <main> <section> <h2>Latest News</h2> <article> <h3>Article Title</h3> <p>Article content goes here.</p> </article> </section> <aside> <h3>Related Links</h3> <p>Sidebar content here.</p> </aside> </main> <footer> <p>&copy; 2025 My Website</p> </footer>

Semantic vs. Non-Semantic Comparison

Non-Semantic (harder to read):

<div id="header">...</div> <div id="nav">...</div> <div id="main">...</div> <div id="footer">...</div>

Semantic (clear and meaningful):

<header>...</header> <nav>...</nav> <main>...</main> <footer>...</footer>
Both examples produce the same visual result, but the semantic version is easier to understand, better for accessibility, and better for SEO.
